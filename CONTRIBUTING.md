# Contributing

Contributions that improve correctness, clarity, reproducibility, or practical coverage are welcome.

## Guidelines

- keep explanations concise and technically accurate
- use current `docker compose` syntax
- never commit secrets or personal credentials
- pin deliberate image versions in reproducible labs
- include cleanup commands in every lab
- explain destructive commands clearly
- test commands before submitting changes
- prefer official Docker documentation as the primary reference

## Suggested process

1. Create a focused branch.
2. Make one logical change.
3. Test all affected commands.
4. Update navigation if adding a document or lab.
5. Open a pull request explaining the change and validation.
