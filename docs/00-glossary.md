# Docker Glossary

| Term | Simple definition |
|---|---|
| Docker | Platform and toolset for building, distributing, and running containers |
| Docker Engine | Client-server application that manages Docker objects |
| Docker daemon | Background service, usually `dockerd`, that handles Docker API requests |
| Docker CLI | The `docker` command-line client |
| Docker Desktop | Desktop package containing Docker Engine access, CLI, Compose, and supporting components |
| Container | Isolated runtime instance of an image |
| Image | Immutable package and metadata used to create containers |
| Dockerfile | Text recipe used to build an image |
| Build context | Files made available to an image build |
| Layer | Reusable, content-addressed filesystem change in an image |
| Registry | Service that stores and distributes images |
| Repository | Collection of related image tags in a registry |
| Docker Hub | Docker's hosted registry service |
| Tag | Human-readable, movable label for an image version |
| Digest | Content-based identifier for exact image content |
| Volume | Docker-managed persistent storage |
| Bind mount | Host path mounted into a container |
| Network | Docker object that connects containers |
| Port publishing | Forwarding traffic from a host port to a container port |
| Compose | Tool and specification for defining multi-container applications |
| Service | Container configuration within a Compose application |
| Healthcheck | Command used to report whether a containerized application is healthy |
| Entrypoint | Main executable configured for a container |
| PID 1 | Main process inside a container |
| Orchestration | Automated deployment, scaling, recovery, and management of containers |
